# Internationalization (i18n) — Design Document

**Date:** 2026-05-07  
**Project:** devinbi-shopify  
**Stack:** next-intl, Next.js 16 App Router, Tailwind CSS v4 RTL, Shopify @inContext

---

## Summary

Four-language storefront (TR, EN, DE, AR) using `next-intl` for UI strings and Shopify's `@inContext` directive for localized product/collection content. Arabic requires RTL layout. URLs are locale-prefixed (`/tr/`, `/en/`, `/de/`, `/ar/`). Missing translations fall back gracefully at both layers.

---

## Supported Locales

| URL locale | next-intl | Shopify language | Shopify country | Direction |
|-----------|-----------|-----------------|----------------|-----------|
| `tr` | `tr` | `TR` | `TR` | LTR |
| `en` | `en` | `EN` | `US` | LTR |
| `de` | `de` | `DE` | `DE` | LTR |
| `ar` | `ar` | `AR` | `SA` | RTL |

**Default locale:** `tr`

## Architecture

```
app/
└── [locale]/                     # tr | en | de | ar
    ├── layout.tsx                 # sets <html lang dir>, loads locale font
    ├── page.tsx
    ├── products/[handle]/
    ├── collections/[handle]/
    ├── search/
    ├── cart/
    └── account/

messages/
├── tr.json                        # UI strings — Turkish (source of truth)
├── en.json
├── de.json
└── ar.json

proxy.ts                           # locale detection + redirect (Next.js 16: middleware renamed to proxy)
i18n/
├── config.ts                      # supported locales, default locale
└── request.ts                     # next-intl server config (getRequestConfig)
```

---

## Routing & Proxy

`proxy.ts` runs on every request (Next.js 16 renamed `middleware.ts` → `proxy.ts`, function `middleware` → `proxy`):
1. Detects locale from URL prefix
2. If no prefix, reads `Accept-Language` header and redirects to matching locale
3. Falls back to default locale (`tr`) if no match

## Translation Structure

All UI strings live in `messages/*.json`. Keys are namespaced by feature:

```json
{
  "nav": {
    "home": "...",
    "collections": "...",
    "search": "...",
    "cart": "...",
    "account": "..."
  },
  "product": {
    "addToCart": "...",
    "outOfStock": "...",
    "price": "...",
    "variants": "..."
  },
  "cart": {
    "empty": "...",
    "total": "...",
    "checkout": "...",
    "remove": "..."
  },
  "account": {
    "login": "...",
    "register": "...",
    "logout": "...",
    "orders": "..."
  },
  "search": {
    "placeholder": "...",
    "noResults": "..."
  }
```

**Usage in Server Components:** `getTranslations({ locale, namespace: 'nav' })`  
**Usage in Client Components:** `useTranslations('nav')`

## Fallback Strategy

### UI strings (next-intl)
`fallbackLocale: 'en'` — if a key is missing in any locale file, English is used. No error thrown.

### Shopify content
Shopify natively falls back to the store's default language when a translation is not defined for the requested `@inContext(language: XX)`. No additional code needed.

## Shopify Integration

`shopifyFetch` accepts an optional `locale` parameter. When provided, the query is wrapped with `@inContext`:

```ts
shopifyFetch({
  query: `query GetProduct($handle: String!) { product(handle: $handle) { title } }`,
  variables: { handle },
  locale: 'de',
  revalidate: 3600,
})

// Becomes:
query GetProduct($handle: String!) @inContext(language: DE, country: DE) {
  product(handle: $handle) { title }
}
```

Page components pass `params.locale` to all Shopify queries. Product titles, descriptions, and collection names are localized by Shopify. UI chrome (buttons, labels, errors) is localized by next-intl.

---

## RTL Support (Arabic)

`app/[locale]/layout.tsx` sets `dir` and `lang` based on locale:

```tsx
const dir = locale === 'ar' ? 'rtl' : 'ltr'
return <html lang={locale} dir={dir}>...</html>
```

Tailwind CSS v4 `rtl:` variants handle layout mirroring throughout components (padding, margin, flex direction, text alignment). Arabic font (`Noto Sans Arabic` or similar) loaded via `next/font` only for `ar` locale.

## What Gets Localized

| Content | Source |
|---------|--------|
| Product titles & descriptions | Shopify `@inContext` |
| Collection names & descriptions | Shopify `@inContext` |
| Prices & currency format | Shopify `@inContext(country: XX)` |
| Navigation labels | `messages/*.json` |
| Button labels, form fields | `messages/*.json` |
| Error messages, empty states | `messages/*.json` |
| Checkout (redirect to Shopify) | Shopify handles natively |
